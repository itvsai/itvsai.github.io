Introduction

HTML is composed of a tree ( i.e elements inside elements ) of HTML elements and other nodes , such as text nodes . Each element can have HTML attributes specified. Elements can also have content, including other elements and text. Many HTML elements represent meaning (semantics). For example, the title element represents the title of the document.

In the HTML syntax, most elements are written with a start tag and an end tag, with the content in between. An HTML tag is composed of the name of the element, surrounded by angle brackets. An end tag also has a slash after the opening angle bracket, to distinguish it from the start tag. For example, a paragraph, which is represented by the p element, would be written like following :

 

<p>P is element p with surrounded brackets is tag slash p(/p) with surrounded brackets is the end tag</p>

Input Types attributes to use like the following
 type="checkbox"
A checkbox. Can be checked or unchecked.
 type="radio"
A radio button. If multiple radio buttons are given the same name, the user will only be able to select one of them from this group.
 type="button"
A general-purpose button. The element <button> is preferred if possible (i.e. if the client supports it) as it provides richer possibilities.
 type="submit"
A submit button.
 type="image"
An image button. The image URL may be specified with the src attribute.
 type="reset"
A reset button for resetting the form to default values.
 type="text"
A one-line text input field. The size attribute specifies the default width of the input in character-widths. max-length sets the maximum number of characters the user can enter (which may be greater than size).
 type="password"
A variation of text. The difference is that text typed in this field is masked — characters are displayed as an asterisk, a dot or another replacement. It should be noted, however, that the password is still submitted to the server as clear text, so an underlying secure transport layer like HTTPS is needed if confidentiality is a concern.
 type="file"
A file select field (for uploading files to a server).
 type="hidden"
hidden inputs are not visible in the rendered page, but allow a designer to maintain a copy of data that needs to be submitted to the server as part of the form. This may, for example, be data that this web user entered or selected on a previous form that needs to be processed in conjunction with the current form.

    display: block;

    
 font-size: 4em;

 margin-bottom: 0.67em;

    margin-left: 0;

    
 margin-right: 0;

    font-weight: bold;

}



what is this ?



h1 called selector , then 7 property with 7 values (i.e display is property and block is a value ) 



When the browser start to show ( render ) any web page it will see the tag style first then it will see the selector h1 and what are inside that selector so when it came to any <h1> tag it will apply what are inside the h1 selector and ignore the real <h1> previous assigned properties from the HTML library developer maker ( DOM library or DOM API , Document Object Model (DOM) , application programming interface (API) ) , all property meaning are very easy to understand for example here we want the font of h1 heading to be with the value 4 em [ 1 em equal for 10 pixels of the screen resolution ] , margin top mean that the distance between words in between h1 start and end tags should be with the value 40 pixels from the top or the last element tag , font weight mean the thickness of the font , display propert mean how is the element  take the width and the height of what inside start and end tags r will it take the the width of the page or the previous container tag width [ i.e  display : block , display : inline ]
